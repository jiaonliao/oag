## 语言设置

**使用中文回复用户**

## 代码风格规范

### 注释规范
- **所有代码注释必须使用英文**
- 注释尽量详细，覆盖关键逻辑与边界

### 代码质量
- 遵循项目现有的代码风格和约定
- 保持代码简洁，避免过度工程化

## Skills 使用规范

### Skills 触发场景
- 涉及框架问题、工具查找、模块选型、配置问题时，优先选择指导类 skill
- 涉及分层代码修改时，按模块选择 API / Service / Repository 等对应 skill
- 涉及国际化 key、远程调用或跨服务接口时，选择对应专项 skill

### Skills 主动触发规则

识别用户意图后主动调用对应 skill，不要等待用户明确要求：
- 涉及多层代码时按需调用多个 skill
- 代码审查时根据代码层级调用对应 skill

### Skills 调用汇总

任务完成时，输出本次任务中调用的所有 skills 列表：
- 格式：`📋 本次调用的 Skills: skill-name-1, skill-name-2, ...`
- 如果未调用任何 skill，输出：`📋 本次调用的 Skills: 无`

## 通用行为准则
- 遵循"足够简单"原则：只做必要的事情
- 保持专业和客观的态度

## 代码修改完成后
- 优先执行与项目匹配的格式化、静态检查与测试命令
- 如果存在 CI 同款校验命令，优先使用 CI 命令进行本地验证
